## Decisions Made

- **Trap insertion + `exec` removal bundled into one commit** (feat, not two). Plan Review Issue #1 was clear that either half alone ships broken behavior; a two-commit split would leave HEAD in a state where the trap is present but non-firing. Single commit preserves logical atomicity of the fix.
- **Test E strategy = E-1 (sed-copy) only, not E-2 (sdk-gated).** Plan 02 acknowledged both; the plan's own verbatim test body was already E-1 throughout. E-2 would have required `python3 -c "import claude_agent_sdk"` preflight, which is not installed in the test env, so it would SKIP everywhere. E-1 PASSes on this host today — a meaningful regression signal.
- **Kept `set -u` (not `-euo pipefail`) in the new test.** Plan 01's stub had `-euo pipefail`, but Test B deliberately drives rc=1 through the trap harness — `set -e` would abort the test at the first harness return. The new header comment documents this trade-off. (This differs from Canary #6's strict-mode convention; test_exit_trap.sh is a unit test suite, not a single assertion.)
- **Mock dialectic.sh emits exactly one JSON line + exit 0, not an empty stdout.** The mock string `{"status":"completed","verdict":"ACCEPT"}` matches Phase 3 D-04 "stdout last-line JSON" contract shape so Test E exercises a realistic happy path. Plan 05 SKILL.md polling will later consume real engine.log tails with the same shape.
